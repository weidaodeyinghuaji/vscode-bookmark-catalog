# Subject
VS Code 插件：Bookmark Catalog（收藏分类搜索导入导出快捷键持久化）

# Introduction
Bookmark Catalog 用于解决开发者在大型代码库中“跳转点难管理、上下文易丢失”的问题。插件通过收藏记录将代码定位信息结构化存储，并提供分类、检索、批量迁移能力。

# Problem Statement
- 仅依赖 VS Code 原生书签能力时，跨项目和跨设备迁移困难。
- 收藏点缺少业务分类与标签，难以形成“任务视图”。
- 查找历史定位点效率低，影响调试与代码评审节奏。

# Goals and Objectives
- 支持收藏新增、分类管理、关键字搜索。
- 支持 JSON 导入导出，满足团队迁移与备份场景。
- 支持快捷键操作与全局持久化。
- 输出可发布、可测试、可持续集成的工程资产。

# User Stories
- 作为前端开发，我希望按“页面/接口/状态管理”分类收藏，以便快速回溯上下文。
- 作为后端开发，我希望通过关键字检索收藏并一键跳转到指定行。
- 作为团队负责人，我希望导出收藏并共享给同组成员复现问题定位路径。

# Technical requirements
- VS Code Extension API（Command、TreeView、Memento）。
- TypeScript 严格模式，单元测试覆盖核心逻辑（分类/搜索/导入导出）。
- JSON 导入需做 schema 校验，防止脏数据污染。
- CI 自动执行 build/test；支持 `.vsix` 打包产物。

# Benefits
- 显著降低上下文切换成本。
- 收藏路径可沉淀为团队资产，提升协作效率。
- 标准化导入导出降低项目迁移成本。

# KPIs
- 收藏新增操作平均耗时 < 5 秒。
- 搜索命中后跳转成功率 >= 99%。
- 核心逻辑单测覆盖率 >= 80%。
- CI 主干构建通过率 >= 95%。

# Development Risks
- 过大导入文件可能导致 UI 卡顿；需限制文件大小并提示分批导入。
- 不同 OS 路径展示差异导致用户理解成本；需统一 URI 展示策略。
- 用户误导入错误格式文件；需严格验证并提供可读错误提示。

# Conclusion
该插件以“可管理、可迁移、可持续演进”为目标，能够在保持使用轻量的前提下，提升中大型项目代码导航效率。后续可演进为团队同步版（云端存储 + 权限控制）。
